# NexusSwarm Security Policy

This document describes the security controls implemented in NexusSwarm and the rules
that apply to all code generated by the AI swarm agents.

---

## Active Security Controls

### 1. Secrets & Environment Variables тЬЕ
- All secrets (NVIDIA API key, DB password) live in `.env` only тАФ never in source code.
- `.env` is listed in `.gitignore` тАФ it is never committed to git.
- Secrets are accessed via `os.environ.get()` / Cloud Secret Manager in production.
- A `.env.example` with empty values is provided as a template.

### 2. Rate Limiting тЬЕ
- All public API endpoints are protected via **slowapi** (`limiter.py`).
- `/submit-task` (AI pipeline): **10 requests/minute/IP**
- `/files/{id}/download`: **5 requests/minute/IP**
- All other endpoints: **60 requests/minute/IP**
- Rate-limited responses return `429 Too Many Requests` with a `Retry-After` header.

### 3. Input Validation & Sanitization тЬЕ
- All user inputs validated server-side with **Pydantic** `Field()` constraints.
- `TaskRequest`: `title` min 1 / max 100 chars; `description` max 2000 chars.
- HTML-escape applied via `html.escape()` before storing or processing.
- `security_utils.sanitize_llm_input()` strips prompt-injection patterns before any LLM call.

### 4. LLM / AI-Specific Security тЬЕ
- **Prompt injection prevention**: all user text is sanitized before forwarding to NIM.
- **Cost attack prevention**: per-IP token budget (50,000 tokens per 15-minute window).
- **Output sanitization**: LLM responses are stripped of `<script>` tags, `on*` handlers,
  and `javascript:` URIs before being stored or returned (`security_utils.sanitize_llm_output()`).
- **max_tokens enforced** on all LLM calls (2048тАУ4096 depending on agent).
- NVIDIA API key is server-side only тАФ never sent to the browser.

### 5. HTTP Security Headers тЬЕ
Applied in `main.py` via custom middleware:
- `Content-Security-Policy` тАФ restricts script/style sources; `frame-ancestors 'none'`
- `X-Frame-Options: DENY` тАФ clickjacking prevention
- `X-Content-Type-Options: nosniff`
- `Strict-Transport-Security` тАФ forces HTTPS
- `Referrer-Policy: strict-origin-when-cross-origin`
- `X-Powered-By` header is removed

### 6. CORS тЬЕ
- No wildcard `*` CORS in production.
- Origins explicitly whitelisted via `ALLOWED_ORIGINS` environment variable.
- Set in `main.py` using `CORSMiddleware` with explicit origin list.

### 7. File Storage Security тЬЕ
- `secure_path()` in `file_storage.py` prevents directory traversal.
- All file paths are resolved to absolute paths and checked against the base directory.
- Files stored in GCS bucket (outside web root) or `/app/workspace` (local).

### 8. Error Handling тЬЕ
- Generic error messages returned to clients ("Something went wrong.").
- Full error context (exception, route, timestamp) logged server-side.
- Stack traces, database schemas, and internal paths are never returned to clients.

### 9. SQL Security тЬЕ
- All database access uses **SQLAlchemy** ORM with parameterized queries.
- No raw string concatenation into SQL queries anywhere.
- DB user has only required permissions (configured in Cloud SQL).

### 10. Dependency Security
Run pip-audit regularly:
```bash
pip install pip-audit
pip-audit -r backend/requirements.txt
```
Or use the provided script:
```bash
python backend/scripts/audit_deps.py
```

## Agent-Generated Code Security

Every AI agent that generates code (BackendAgent, FrontendAgent, APIAgent, DeployAgent, TestAgent)
has the 12-point security checklist injected into its system prompt via
`agents/agent_security_prompt.py тЖТ get_secure_system_prompt()`.

This means **every website, API, and application built by the swarm must follow the same
security rules as the NexusSwarm backend itself**.

The rules enforced in generated code:
1. No hardcoded secrets тАФ env vars only
2. Rate limiting on all public endpoints (auth: 5/15min, general: 60/min, AI: 10/min)
3. Server-side input validation with Pydantic / Zod
4. bcrypt/argon2 password hashing (min cost 12), JWT with short expiry
5. ORM / parameterized queries only тАФ no raw SQL concatenation
6. Explicit CORS origins (no wildcard)
7. Security headers (CSP, HSTS, X-Frame-Options: DENY, etc.)
8. File upload validation (MIME + extension + UUID rename + cloud storage)
9. Generic error messages to clients, full detail in server logs
10. No `dangerouslySetInnerHTML` / `eval()` / raw innerHTML in frontend
11. LLM calls server-side only, with max_tokens + input sanitization
12. Pre-deploy checklist enforcement

## Pre-Deploy Checklist

Before every deploy, verify:

- [ ] `.env` is NOT committed to git
- [ ] All secrets are set in Cloud Secret Manager / hosting env vars
- [ ] Debug mode and dev logging are OFF
- [ ] Database is NOT publicly exposed
- [ ] HTTPS is enforced (Cloud Run enforces this automatically)
- [ ] Rate limiting is active on all public endpoints
- [ ] CORS is restricted to known origins
- [ ] `pip-audit` shows no HIGH/CRITICAL issues
- [ ] Security headers are present (verify with `curl -I <url>`)
